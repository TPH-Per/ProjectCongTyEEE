**3 file thay đổi:**

- Bookings hôm nay tự động hiện trong right panel, không cần user chọn date

#### 1.3 `customerApi.ts` — Mock fallback khi Supabase không cấu hình

- Import `isSupabaseConfigured` từ `@/lib/supabase`
- **`getAreas()`**: Trả 5 khu mock (A, B, C, R, T) với `id` dạng `zone-a`, `zone-b`, ...
- **`getTables(areaId)`**: Map zone → danh sách bàn thực tế (A01-A09, B01-B03, C01-C08, R01-R08, T01-T08), status `occupied` cho A03/A04/A05, `available` cho các bàn còn lại
- **`getRawMenuItems()`**: Walk `menuTemplate` (categories → subcategories → items) và generate UUID ổn định dạng `00000000-0000-4000-8000-000000000001`, `...002`, ... cho mỗi món. Điều này đảm bảo:
  - `isValidUUID()` pass trong cart validation
  - `loadMenu()` substitution hoạt động (tìm thấy DB item theo tên → thay mock ID bằng UUID thật ảo)
  - Không còn lỗi "Món ăn không hợp lệ" khi chạy offline

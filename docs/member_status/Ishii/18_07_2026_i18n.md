## 🎯 TỔNG QUAN

Triển khai tính năng chuyển đổi ngôn ngữ (Tiếng Việt / English / 日本語) cho toàn bộ màn hình khách hàng. Project **đã có sẵn** hạ tầng vue-i18n v9.14.5 với 3 file locale (vi/en/ja), component `LanguageSwitcher`, và helper `setApplicationLanguage()`. Nhiệm vụ chính là bổ sung translation keys cho namespace `customer` và wire up toàn bộ text hardcode tiếng Việt trong các component customer sang `$t()`.

### Vấn đề quan trọng phát hiện & khắc phục

`CustomerLayout.vue` có language switcher riêng (3 cờ 🇻🇳 🇬🇧 🇯🇵) nhưng **chỉ set `store.currentLanguage`** — một state cục bộ trong `customerStore` — mà **không gọi** `setApplicationLanguage()`. Điều này có nghĩa:
- vue-i18n global locale **không thay đổi** khi user bấm cờ
- `<html lang>` **không cập nhật**
- localStorage `ngu-cat.locale` **không lưu**
- Tất cả `$t()` calls trả về tiếng Việt bất kể user chọn ngôn ngữ nào

**Đã sửa:** Gọi `setApplicationLanguage()` đồng bộ vue-i18n global + `<html lang>` + cả 2 Pinia stores (`useI18nStore` + `useLanguageStore`) + localStorage.

### ✅ 2. SỬA LANGUAGE SWITCHER TRONG CustomerLayout.vue

**Trước:**
```js
function setLanguage(lang) {
  store.currentLanguage = lang  // ❌ Chỉ set local state, i18n không sync
  store.addNotification(`Đã chuyển sang ngôn ngữ: ${lang === 'vi' ? 'Tiếng Việt' : ...}`)
}
```

**Sau:**
```js
import { setApplicationLanguage } from '@/helpers/i18n'
import { useI18n } from 'vue-i18n'

const { locale, t } = useI18n()

function setLanguage(lang) {
  setApplicationLanguage(lang)  // ✅ Sync vue-i18n + html lang + localStorage + both stores
  const langName = lang === 'vi' ? 'Tiếng Việt' : lang === 'en' ? 'English' : '日本語'
  store.addNotification(t('customer.languageChanged', { lang: langName }), 'info')
}
```

Flag highlighting đổi từ `store.currentLanguage === 'vi'` → `locale === 'vi'` (đọc từ global vue-i18n ref).

**Toàn bộ text hardcode trong CustomerLayout** cũng được wire up:
- `placeholder="Tìm kiếm..."` → `:placeholder="$t('customer.header.search')"`
- `title="Giỏ hàng"` → `:title="$t('customer.cart.title')"`
- `title="Theo dõi món ăn"` → `:title="$t('customer.tracking.title')"`
- `title="Gọi phục vụ"` → `:title="$t('customer.serviceRequest.title')"`
- `title="Lịch sử gọi món"` → `:title="$t('customer.orderHistory.title')"`
- `title="Thoát bàn"` → `:title="$t('customer.exitTable.confirmTitle')"`
- Swal dialogs trong `handleExitTable()` dùng `t()`
- `toLocaleTimeString('vi-VN')` → `toLocaleTimeString(timeLocale.value)` locale-aware

## 🔧 KIẾN TRÚC I18N HIỆN TẠI (SAU THAY ĐỔI)

```
User bấm cờ 🇻🇳/🇬🇧/🇯🇵 trong CustomerLayout
    ↓
setApplicationLanguage(lang)  ← helpers/i18n.ts
    ├── useLanguageStore.setLanguage(lang)     ← Pinia store 1 (app_lang)
    ├── useI18nStore.setLocale(lang)           ← Pinia store 2 (ngu-cat.locale)
    │     └── setI18nLocale(lang)              ← vue-i18n global.locale.value = lang
    ├── document.documentElement.lang = lang   ← HTML attribute
    └── document.documentElement.dataset.locale = lang
    ↓
Tất cả $t('customer.*') trong template tự động re-render bằng ngôn ngữ mới
Tất cả t('customer.*') trong script trả về text đúng ngôn ngữ
```

## 5. SỬA LỖI TRÙNG LẶP CODE — ReceptionOrderView

**File:** `src/views/reception/ReceptionOrderView.vue`

### Vấn đề
Lỗi `[vue/compiler-sfc] Identifier 'tableActions' has already been declared`. Composable `useTableOperations()` được trích xuất nhưng code gốc không được dọn dẹp, gây trùng lặp định danh.

### Giải pháp
Xóa toàn bộ code trùng lặp (khoảng 900 dòng):

| Code bị xóa | Lý do |
|-------------|-------|
| `allTableActions` array + `tableActions` computed | Đã có trong composable |
| `getTablesWithStatus` | Không được sử dụng |
| `handleTableAction` function | Đã có trong composable |
| `validTargetTables` computed + `isValidTargetTable` | Đã có trong composable |
| `handleTableClickInSelectionMode` + `executeTransfer/Merge/SplitOrder` | Đã có trong composable |
| `toggleItemForSplit`, `changeSplitQty`, `splitItemTotal`, `closeSplitItemModal`, `executeSplitItem` | Đã có trong composable |
| `executeCancel` | Đã có trong composable |
| `exitSelectionMode`, `cancelSelectionMode`, `getSelectionModeTitle`, `handleTableClick`, `closeContextMenu` | Đã có trong composable |
| `handleTableListActionClick` | Đã có trong composable |

**Giữ lại:** Các hàm unique không nằm trong composable — `handleRemoveOrCancelItem`, `openCancelItemModal`, `pressCancelPinDigit`, `backspaceCancelPin`, `executeCancelItem`, `generateCancelItemEscPos`.

## 6. SỬA LỖI CUSTOMER FLOW — Không vào được Menu + Enter key

### 6.1. Lỗi Enter key quay về Passcode

**File:** `PasscodeInput.vue`, `CustomerHome.vue`, `SelectArea.vue`, `SelectTable.vue`, `SelectBranch.vue`, `AreaGrid.vue`, `BranchGrid.vue`, `TableGrid.vue`

**Nguyên nhân:** Tất cả `<button>` thiếu `type="button"`. HTML mặc định `<button>` là `type="submit"` — nhấn Enter trigger button đầu tiên có focus (nút "lock" gọi `resetSetup()` → về passcode).

**Fix:** Thêm `type="button"` vào tất cả button trong 8 file customer flow.

### 6.2. Lỗi không vào được Menu sau khi chọn bàn

**File:** `src/views/customer/CustomerHome.vue`

**Nguyên nhân:** Khi Supabase được cấu hình, `selectTable('A01')` query DB với `id = 'A01'` (không phải UUID) → query trả null → `store.selectedTable` không được set → `confirmTable()` thoát sớm → không tạo session → `router.push` chạy nhưng `CustomerLayout` hiển thị layout "không có session".

**Fix:**
1. `handleTableSelect` — Set `store.selectedTable = table` trực tiếp từ tham số, không phụ thuộc API
2. `handleTableConfirm` — Fallback session: nếu `store.session` vẫn null sau `confirmTable()`, tạo local session từ `store.selectedTable`, lưu localStorage, rồi `router.push`

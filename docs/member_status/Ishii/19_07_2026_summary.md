### 3.4. Pricing Rules (chia sẻ với cashier)

| Rule | Mô tả |
|------|-------|
| Package pricing | Món trong gói buffet → `price = 0` ("0K — Trong gói") |
| Lunch 50% | Giảm 50% giá, hiển thị "{half}đ (Lunch 50%)" |
| Service charge | 5% × subtotal |
| VAT | 8% × (subtotal + service charge) |
| Engine | `@/utils/packageRules` → `computeTotals()` — đồng nhất customer & cashier |

### 3.5. Session Persistence

- **localStorage keys:** `nguucat_customer_session`, `nguucat_customer_auth`, `nguucat_customer_table`
- `restoreSessionFromLocalStorage()` — gọi trong `CustomerLayout` onMounted
- `clearSession()` — xóa localStorage + reset store

## 7. ⚠️ VẤN ĐỀ KIẾN TRÚC CẦN LƯU Ý

### 7.1. Khách order từ tablet KHÔNG flow vào Reception

```
Customer Tablet                       Reception
┌──────────────────────┐            ┌──────────────────────┐
│ confirmOrder()       │            │ pendingOrders =      │
│ → supabase.rpc(      │───────❌───│   [MOCK A04,A08,B02] │
│   create_self_service│  KHÔNG    │                      │
│   _order)            │  KẾT NỐI  │ fetchDbTables() →    │
│                       │           │   chỉ load 1 lần     │
│ Order vào DB ✓       │           │                      │
└──────────────────────┘            └──────────────────────┘
```

| Vấn đề | Chi tiết |
|--------|----------|
| Không realtime | Không có `supabase.channel()`. Order từ customer không tự xuất hiện ở Reception |
| `pendingOrders` là mock | 3 đơn hardcoded, không đọc từ DB |
| Không poll | Không có `setInterval` fetch orders mới |
| Transfer/merge in-memory | Chỉ sửa `restaurantStore` trong RAM, không ghi DB |
| Chỉ cancel có DB | `supabase.rpc("hall_cancel_order_or_item")` — duy nhất có backend |

### 7.2. Flow đúng cần có

```
Customer Tablet                     Reception
┌──────────────────────┐            ┌──────────────────────┐
│ confirmOrder()       │            │ Realtime subscription │
│ → RPC insert order   │──→ DB ────→│ .on('INSERT', orders) │
│                       │            │ → toast "Đơn mới!"   │
│                       │            │ → pendingOrders.push │
│                       │            │ → table đổi màu đỏ   │
└──────────────────────┘            └──────────────────────┘
```

## 9. KẾT LUẬN

Hệ thống POS Ngưu Cát hiện có **2 giao diện hoàn thiện về UI**:

1. **Reception Order** (`/reception/order`): Đầy đủ 4 tab (sơ đồ bàn, thực đơn, hóa đơn, chưa xử lý), 6 thao tác bàn (chọn món, chuyển, ghép, tách phiếu, tách món, hủy), tính tiền (service 5% + VAT 8%).

2. **Customer Self-Service** (`/customer`): Đầy đủ 10 màn hình (passcode → khu vực → bàn → menu → giỏ → order → lịch sử → phục vụ → đánh giá → kết thúc), 9 loại yêu cầu dịch vụ, theo dõi món real-time (mock).

**Khoảng cách lớn nhất cần khắc phục:** Thiếu realtime sync giữa Customer tablet và Reception — order từ khách không tự xuất hiện ở màn hình lễ tân. Cần bổ sung Supabase Realtime subscription hoặc polling mechanism.

## V. PHÂN TÍCH & HOÀN THIỆN NGHIỆP VỤ QUẢN LÝ KHO (WAREHOUSE WORKFLOWS)

Để đảm bảo tính toàn vẹn của chuỗi cung ứng nội bộ nhà hàng, tôi đã thực hiện phân tích chuyên sâu và so sánh đối chiếu 3 quy trình quản lý kho cốt lõi, từ đó phát hiện các điểm thiếu sót và tạo mới các tài liệu workflow tương ứng.

### 1. Phân tích So sánh 3 Quy trình Kho

Tiến hành so sánh chi tiết giữa **Receipt (Nhập kho)**, **Inventory (Tồn kho)** và **Reconciliation (Đối soát cuối tháng)** theo 7 tiêu chí: số bước kiểm tra, phân loại hàng, vị trí trong kho, dòng trống phân cách, phương thức đối chiếu, xử lý chênh lệch và vòng lặp xử lý.

**Kết quả phát hiện:**

- **Receipt:** Đơn giản nhất (5 bước kiểm tra), tập trung vào tính hợp lệ của hóa đơn.
- **Inventory:** Phức tạp hơn (8 bước), có thêm logic di chuyển dòng và chèn dòng trống phân cách theo nhóm nhà cung cấp.
- **Reconciliation:** Phức tạp nhất (10+ bước), có vòng lặp xử lý và 3 mức chênh lệch (Khớp/Nhỏ/Lớn).

### 2. Tạo mới Tài liệu Workflow Kho

- [receipt_workflow.mmd](file:///E:/Công%20ty/Task/260/Product/docs/workflows/restaurant_business_process/05_inventory_procurement/receipt_workflow.mmd): Quy trình tiếp nhận hóa đơn từ nhà cung cấp, kiểm tra chữ ký, scan và nhập liệu.
- [inventory_workflow.mmd](file:///E:/Công%20ty/Task/260/Product/docs/workflows/restaurant_business_process/05_inventory_procurement/inventory_workflow.mmd): Quy trình cập nhật vị trí, phân nhóm nhà cung cấp và quản lý dòng trống trong sổ kho.
- [reconciliation_workflow.mmd](file:///E:/Công%20ty/Task/260/Product/docs/workflows/restaurant_business_process/05_inventory_procurement/reconciliation_workflow.mmd): Quy trình đối soát hóa đơn cuối tháng với logic tự động/thủ công và xử lý chênh lệch.

## VIII. XÂY DỰNG DESIGN SYSTEM CONTEXT CHO KITCHEN MODULE

Để chuẩn hóa giao diện Kitchen Display System (KDS) - vốn có yêu cầu đặc thù khác biệt so với POS thông thường, tôi đã biên soạn tài liệu **Design System Context** hoàn chỉnh.

### 1. Nguyên tắc Thiết kế Đặc thù cho Bếp

- **Đọc được từ xa 2-3m:** Font size tăng 20-30% so với POS.
- **Thao tác với găng tay:** Touch targets tối thiểu 56x56px.
- **Môi trường ánh sáng mạnh:** Dark mode mặc định, contrast ratio ≥ 4.5:1.
- **Phân cấp thông tin rõ ràng:** Color coding cho 6 trạng thái ticket.

### 2. Bảng màu Trạng thái Ticket (Status Colors)

| Trạng thái         | Hex Code  | Ứng dụng               |
| ------------------ | --------- | ---------------------- |
| New (Mới)          | `#1A237E` | Ticket vừa nhận từ POS |
| Cooking (Đang nấu) | `#E65100` | Bếp đang chế biến      |
| Ready (Sẵn sàng)   | `#2E7D32` | Chờ Staff lấy món      |
| Delayed (Trễ)      | `#C62828` | Quá thời gian chuẩn    |
| Priority (Ưu tiên) | `#6A1B9A` | Remake/VIP             |
| Cancelled (Hủy)    | `#424242` | Món bị hủy             |

### 3. Typography & Spacing System

- **Ticket Number:** 32px Bold (đọc từ xa)
- **Table Number:** 24px Bold
- **Timer Display:** 28px Bold (Courier New - monospace)
- **Item Name:** 18px Semibold
- **Action Buttons:** 16px Semibold, min-height 56px
- **Gap giữa các ticket:** 16px
- **Border radius:** 12px (ticket cards)

### 4. Animation & Alert System

- **Pulse animation:** Cho priority/delayed tickets (2s loop)
- **Blink animation:** Cho urgent alerts (0.5-1s loop)
- **Slide-in animation:** Khi ticket mới xuất hiện
- **Pulse-border animation:** Viền nhấp nháy cho ticket trễ (>15 phút)

### 5. Responsive Breakpoints

- **>1920px:** 5-6 tickets/row
- **1366-1920px (POS chuẩn):** 4 tickets/row
- **768-1366px (Tablet):** 2-3 tickets/row
- **<768px (Mobile):** 1-2 tickets/row

## IX. THIẾT KẾ UI CHI TIẾT CHO KITCHEN MODULE

Dựa trên Design System, tôi đã xây dựng bộ thông số kỹ thuật UI chi tiết cho từng component của Kitchen Display System, sẵn sàng bàn giao cho đội ngũ phát triển.

### 1. Ticket Card Component

- **Kích thước:** Width 280px (fixed), Min-height 220px
- **Cấu trúc:** Header (Ticket #, Timer) → Table Info → Item List → Notes → Action Buttons
- **Hover effect:** `translateY(-4px)` + shadow tăng
- **Priority state:** Border 3px tím + pulse animation
- **Delayed state:** Background gradient đỏ + blink animation

### 2. Ticket Detail Modal

- **Kích thước:** Width 600px max, Height 80vh max
- **Sections:** Thông tin cơ bản → Danh sách món (với allergy highlight) → Timeline → Ghi chú đặc biệt → Action buttons
- **Allergy highlight:** Nền đỏ nhạt, border đỏ, icon cảnh báo

### 3. Filter & Status Bar

- **Filter Bar (60px):** Station dropdown, Status dropdown, Sort, Search, Sound toggle
- **Status Bar (50px):** 4 counters (New/Cooking/Ready/Delayed) với màu tương ứng

### 4. Accessibility Requirements

- **Focus states:** `outline: 3px solid #2196F3` cho tất cả interactive elements
- **High contrast mode:** Border width tăng lên 3px
- **Reduced motion:** Disable animations khi `prefers-reduced-motion: reduce`
- **Touch targets:** Tối thiểu 48x48px (56x56px cho kitchen)

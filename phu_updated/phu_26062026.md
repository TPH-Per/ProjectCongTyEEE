## 4. Nguyên tắc bảo mật đã tuân thủ

- **Không hardcode production password** trong code: passwords trong docs là dev-only, có cảnh báo rõ.
- **Không dùng service_role key trong frontend**: chỉ dùng anon key + RLS.
- **Không bypass RLS**: mọi query từ client qua anon key đều chịu RLS.
- **Không dùng `user_metadata` cho authorization**: chỉ dùng `app_metadata.role` (do custom access token hook inject từ `public.users.role`).
- **Không đổi enum DB**: enum hiện tại (`order_status`, `table_status`, `payment_method`, `revenue_type`, `shift_status`) được giữ nguyên; frontend fix theo enum có sẵn.
- **Edge Function branch ownership check**: `checkout/index.ts` check `profile.branch_id === order.branch_id` cho non-admin.
- **Không gọi payment gateway thật**: payment methods vẫn mô phỏng (cash/card/transfer/voucher).
- **Không gọi API bên thứ ba**.
